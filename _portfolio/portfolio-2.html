Project overview

tidypolis is an R package designed to streamline the process of cleaning and preparing global polio surveillance data for analysis. The package provides a set of functions that facilitate downloading global polio data from the WHO Polio Information System (POLIS) API, as well as perform data quality checks, archival, and cleaning. It also has a caching system that works locally and on Azure storage containers to optimize data retrieval and minimize redundant downloads. Some of the key features of the package include:

  • Functions to download and cache global polio surveillance data from the POLIS API.
  • Data cleaning functions to standardize variable names, formats, and values.
  • Data quality checks to identify and handle missing or inconsistent data.
  • Functions to archive cleaned data for future reference and reproducibility.
  • Compatibility with Azure storage containers for efficient data management in cloud environments such as Posit Workbench.
  • Cleaning of WHO geodatabase based on methodologies used by the CDC Surveillance, Innovation, and Research team within the Polio Eradication Branch.
The tidypolis package provides a standardized and efficient workflow for cleaning global polio surveillance data and also increases transparency and reproducibility in data analysis. Nishant Kishore, PhD, was the original lead developer of the package, and Nicholas Heaghney, MS, was its primary maintainer up until May 2025. Both Nishant and Nick had worked on much of the data cleaning code found in the package. I have since taken over maintenance and further development of it. My key contributions include:
  • Implementing the system to work on Posit Workbench.
  • Improving data cleaning functions to handle edge cases and inconsistencies in the data.
  • Enhancing documentation and vignettes to improve user understanding and adoption of the package.
  • Streamlining code to improve readability and performance through modularization.
When I took over as the primary maintainer, my primary goal was to make sure the package runs more efficiently and be easier to maintain. In particular, I was the project lead for the modularization effort of the data cleaning script to improve code readability and maintainability, especially for collaborators outside of CDC. This was achieved by breaking down the monolithic cleaning script into smaller, more manageable functions that each perform a specific task. The modularization effort was done in collaboration with Nicholas Heaghney, MS (CDC), Nishant Kishore,PhD (CDC), and Mohammed Yusuf, PhD (WHO).
